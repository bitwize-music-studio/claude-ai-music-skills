## Promo Directory Feature

Social media copy now has its own directory structure in each album:

```
{album}/promo/
├── campaign.md   # Strategy, schedule, key messages
├── twitter.md    # Tweets/threads per track
├── instagram.md  # Captions + hashtags per track
├── tiktok.md     # Captions per track
├── facebook.md   # Posts per track
└── youtube.md    # Descriptions per track
```

### What Changed

- **SOURCES.md template**: The "Social Media Copy" section now points to `promo/` instead of containing inline copy
- **New albums**: `/new-album` automatically creates `promo/` and copies templates
- **Release workflow**: Pre-release QA now checks that `promo/` copy is populated
- **Promo workflow**: Next steps updated to reference `promo/` templates

### What Didn't Change

- `promo_videos/` in `{audio_root}` — video files stay in the audio directory
- Existing social media copy in SOURCES.md files — not automatically migrated (user can copy manually if desired)

### For Existing Albums

The auto action creates the `promo/` directory in any album that doesn't have one. The confirm action copies the blank templates into those directories. You can then fill them in at your own pace — they're templates, not requirements.
